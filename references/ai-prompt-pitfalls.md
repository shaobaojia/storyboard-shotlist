# AI 生图/生视频提示词避坑

> 当分镜表用作 Seedance 等 AI 视频模型的提示词来源时，动作调度列的写法直接影响生成质量。
> 以下坑均来自实际踩坑经验。

## 坑 1：画外目标物入画

**问题：** AI 模型读到动作调度里的物体名称，即使该物体在画外，也会强行塞进画面。

**错误写法：**
```
他看着电玩城门口。眼神从空变成定。
```
→ AI 必在画面里塞一个电玩城门口——哪怕这是正打镜头，电玩城应该在角色背后。

**正确写法：**
```
眼神虚焦——看着镜头方向，瞳孔没有对准任何东西。
```
→ 只描述角色的表演（视线方向、眼神状态），不提画外目标。

**原则：** 动作调度描述角色**做了什么**，不描述角色**看到了什么**。画外物体名一个都不准出现。

## 坑 2：地上的人/物入画

**问题：** 动作调度里写"地上的男人"，AI 会往画面里画一个地上的男人。

**错误写法：**
```
柯基隔着玻璃，瞥向地上瘫倒的男人。
```
→ AI 在画面里画一个瘫倒的男人。

**正确写法：**
```
柯基隔着玻璃，瞥向镜头方向。
```
→ 空间关系列已交代男人在画外，动作调度只管柯基的表演。

## 坑 3：光线反射命名目标

**问题：** 写"霓虹灯的红蓝光轮流打在脸上"——AI 可能把霓虹灯招牌画进画面。

**处理：** 光影信息放导演备注（给人看的），动作调度不提光源名称。或写"画外彩光"替代"霓虹灯光"。

## 通用避坑原则

| 列 | 可以写 | 不能写 |
|:---|:---|:---|
| 空间关系 | 画内元素的位置标签 | 画外元素的名称（除非明确标 `[画外]`） |
| 动作调度 | 角色的肌肉动作、表情、视线方向 | 角色看到的具体目标物名称 |
| 导演备注 | 光源类型、色彩、拍摄条件 | —（备注不喂给 AI） |

## 斜角正打的视线交代

当角色正打镜头需要看向画外目标时，用视线方向替代目标物名称：

```
动作: 视线掠过镜头旁，被画外的光吸引。
备注: 斜角15°偏轴正打。画外霓虹灯光源。
```

→ 表演指导在动作列，技术信息在备注列，AI 只吃动作列。
